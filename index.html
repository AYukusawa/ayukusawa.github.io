Attack Scenarios

1. Session Hijacking: Attacker embeds app in iframe on malicious site. Sends OKTA_TOKEN via postMessage to overwrite victim's Okta token with attacker's, hijacking API requests.


2. User Impersonation: Attacker sends SET_MASQUERADING_COOKIE to set victim's MasqueradedId header to another user's ID, accessing their data.


3. Stored XSS via AI: If attacker can manipulate AI responses (prompt injection), malicious HTML like <img src=x onerror=alert(document.domain)> executes in victim's browser because dangerouslySetInnerHTML has no sanitizer.
